# ESP32 AIoT 即時戰情室 - 操作手冊

本專案提供了一個高質感卻極度輕巧的物聯網展示平台。後端資料庫與前端視覺化儀表板皆能在您的電腦上完全單機離線運行。硬體 (ESP32) 部分則依照環境，提供「有 WiFi (無線)」與「無 WiFi (實體傳輸線)」兩種使用模式。

## ⚙️ 零、基礎軟體伺服器準備 (必要)

不論您之後要插線還是用 WiFi，您電腦上的 API 伺服器與儀表板都必須先被啟動。
請開啟 PowerShell 終端機，並先切換到本專案目錄下 (`cd c:\Users\user\Desktop\esp32_dht11`)：

1. **確保安裝依賴套件 (首次執行才需要)**：
   ```powershell
   python -m pip install -r requirements.txt
   python -m pip install pyserial
   ```
2. **啟動負責儲存與接收的前線：後端資料庫 API**：
   ```powershell
   python api.py
   ```
   *(啟動後，請將這個黑色視窗縮小，保持它在背景獨立運作)*
3. **啟動負責呈現的後勤：視覺化互動儀表板**：
   請開一個 **全新** 的 PowerShell 視窗，位於同樣的目錄下並執行：
   ```powershell
   python -m streamlit run app.py
   ```
   *(程式跑完後，會自動在您的瀏覽器彈出 `http://localhost:8501`)*

待兩者成功運行後，請選擇以下其中一種硬體模式進行資料上傳。

---

## 📶 模式一：有 WiFi 環境 (推薦)
此模式下，ESP32 將能擺脫線材舒服 (只要接上行動電源或電池即可)，透過 WiFi 無線將溫濕度拋轉給您的電腦伺服器。

### 硬體設定與執行步驟：
1. **設定程式**：使用 Arduino IDE 打開本資料夾中的 **`ESP32_AIoT.ino`**。
2. **修改網路參數**：
   找尋程式碼開頭的 WiFi 設定區塊，填入您的 WiFi 帳號與密碼：
   ```cpp
   const char* ssid = "您的WiFi名稱";
   const char* password = "您的WiFi密碼";
   ```
3. **修改本機 IP 指向**：
   按 `Win + R` 輸入 `cmd` 開啟命令提示字元，輸入 `ipconfig` 以查詢您電腦現在的 **IPv4 位址** (Ex: 192.168.1.15)。
   然後替換這一行的 IP：
   ```cpp
   const char* serverName = "http://192.168.1.15:5000/sensor"; 
   ```
4. **燒錄與檢視**：
   * 將 ESP32 接上電腦，點擊上傳燒錄。
   * 燒錄完成後，**可以開啟** Arduino 的 Serial Monitor 看它是否成功連上 WiFi 並開始傳送 (出現 HTTP Response code: 201 代表成功送到後端)。
   * 此刻您的 Streamlit 網頁儀表板上，就會自動浮現包含真實 WiFi 訊號強度的即時溫濕度曲線了！

---

## 🔌 模式二：無 WiFi 環境 (實體線串列埠降級模式)
若當下沒有 WiFi 基地台，您可以直接讓那條插在電腦上的 USB 傳輸線，兼作純資料的傳輸橋樑，將數據離線送進網頁儀表板。

### 硬體設定與執行步驟：
1. **無腦燒錄專用程式**：使用 Arduino IDE 打開 **`ESP32_AIoT_Serial.ino`**，直接點擊上傳燒錄進 ESP32 中（不需要設定密碼與 IP 等任何資訊）。
2. **⚠️ 極度重要：關閉 Serial Monitor**：
   燒錄完成後，**請立刻關閉 Arduino IDE 的 Serial Monitor**。如果沒關掉它，待會 Python 會因為遇到「通訊埠被佔用」而崩潰報錯！
3. **設定 Python 橋接器**：
   使用筆記本開啟專案資料夾中的 **`serial_to_api.py`**，找到並確認您的 COM 埠號是不是符合當前 ESP32 的埠號（預設為 COM4）：
   ```python
   PORT = 'COM4' # 若 ESP32 目前插在 COM3，請改成 'COM3'
   ```
4. **啟動橋接器**：
   在電腦上開啟**第三個全新的 PowerShell 視窗**，執行：
   ```powershell
   python serial_to_api.py
   ```
   * 這個程式會死盯著 COM 埠，一收到 ESP32 的 JSON 字串，就會瞬間把它無縫 POST 給 `api.py`。
   * 您會看到終端機印出「👉 成功轉發至儀表板資料庫！」。為了區別硬體目前的狀態，此時您的網頁儀表板上的 WiFi 訊號名稱會顯示為 `OFFLINE_SERIAL`，訊號強度為 0 dBm。

## 💻 附錄：沒有開發板怎麼辦？ (純軟體模擬測試)
如果您手邊剛好沒有 ESP32 開發板，但又想打開儀表板欣賞炫酷的平滑折線圖。只要開啟另外一個終端機執行模擬器：
```powershell
python esp32_sim.py
```
這個官方模擬器就會直接充當一台 ESP32，每隔 5 秒自動發送一次亂數產生的溫濕度與浮動的網路資料，完美展現您的前端火力。
